# 04. サービスの起動

## 前提条件

- [ ] `01_Tailscale設定.md` 完了
- [ ] `02_APIキーと.env設定.md` 完了（少なくとも `DB_PASSWORD`・`GEMINI_API_KEY`・`N8N_PASSWORD` が設定済み）

---

## ステップ1: 初回セットアップスクリプトの実行

```bash
cd ~/dotfiles/server
bash scripts/setup.sh
```

このスクリプトが以下を自動実行します：
- ディレクトリ確認
- Sambaユーザー設定（パスワード入力を求められます）
- Docker Composeで全サービス起動

---

## 手動で起動する場合

```bash
cd ~/dotfiles/server
sudo docker compose up -d
```

---

## 起動状態の確認

```bash
sudo docker compose ps
```

以下の状態になっていればOK：

```
NAME                    STATUS
immich_server           running
immich_microservices    running
immich_machine_learning running
immich_redis            running
immich_postgres         running
samba                   running
openwebui               running
n8n                     running
```

---

## 各サービスへのアクセス

Tailscale接続後、以下のURLにアクセス：

### Immich（写真管理）
```
http://kokko-server-pavilion:2283
```
初回アクセス時に **管理者アカウント作成** 画面が出ます。
詳細は `05_Immich初期設定.md` を参照。

### OpenWebUI（AIチャット）
```
http://kokko-server-pavilion:3000
```
初回アクセス時にアカウント作成を求められます（ローカルアカウントでOK）。
作成後、Gemini API経由でAIと会話できます。

### n8n（ワークフロー自動化）
```
http://kokko-server-pavilion:5678
```
`.env` で設定した `kokko` / `N8N_PASSWORD` でログイン。
詳細は `06_n8n設定.md` を参照。

---

## Butler Bot の起動（Discord設定後）

`03_Discord_Bot作成.md` 完了後：

```bash
cd ~/dotfiles/server
sudo docker compose --profile butler up -d butler-bot
```

---

## ログの確認

```bash
# 全サービスのログ
sudo docker compose logs -f

# 特定サービスだけ
sudo docker compose logs -f immich-server
sudo docker compose logs -f butler-bot
sudo docker compose logs -f n8n
```

---

## サービスの再起動・停止

```bash
# 特定サービスだけ再起動
sudo docker compose restart openwebui

# 全サービス停止
sudo docker compose down

# 全サービス停止＋ボリューム削除（データも消えるので注意！）
# sudo docker compose down -v  ← 絶対に間違えて実行しないこと
```

---
