# 11. 執事エージェントのホスト操作権限を有効化する

Discordの執事Botに、このサーバーホスト自体を操作させる（シェルコマンド実行・ファイル読み書き）機能を
追加しました。設計の詳細は`plan/11_agent-control.md`を参照。ここでは有効化の手順だけをまとめます。
**この手順はあなた自身のDiscordアカウントの情報が必要なため、代わりに実行できません。**

## 前提

- 破壊的な操作（削除・再起動・設定変更など）は、実行前にDiscordの✅/❌リアクションで承認が必要です
  （5分でタイムアウトすると却下扱い）。
- `DISCORD_OWNER_ID`を設定するまでは、この機能は完全に無効（今まで通りの動作）です。

## 手順

### 1. 自分のDiscordユーザーIDを取得

1. Discordの設定 →「詳細設定」→「開発者モード」をON
2. 自分のアイコンを右クリック →「ユーザーIDをコピー」

### 2. `.env`に追記

```bash
cd ~/dotfiles/server
nano .env
```

```
DISCORD_OWNER_ID=コピーしたユーザーID
```

### 3. BotにAdd Reactions権限を付与

承認フローはDiscordのリアクションを使うため、Bot作成時（`03_Discord_Bot作成.md`）の招待には
含めていなかった権限が必要です。

1. Discordサーバーの設定 →「ロール」
2. Botのロール（Botの名前と同じもの）を選択
3. 「リアクションの追加（Add Reactions）」権限をON

### 4. Botを再起動して反映

```bash
cd ~/dotfiles/server
sudo docker compose up -d butler-bot
```

### 5. 動作確認

`#butler-chat`で試します：

- 読み取り系（すぐ実行されるはず）:「dockerのコンテナ一覧を見せて」「ディスクの空き容量は？」
- 変更系（✅/❌の確認が挟まるはず）:「n8nコンテナを再起動して」

`/status`で「エージェントモード: 有効」と表示されれば設定は反映されています。

## 何かあったら

- ホスト側の実行エージェント（`agent_executor.py`）の状態: `sudo systemctl status butler-agent.service`
- ホスト側の実行エージェントのログ: `sudo journalctl -u butler-agent.service -f`
- 実行履歴（監査ログ）: `cat /mnt/data/ai/context/agent_audit.jsonl`
- 無効化したい場合: `.env`の`DISCORD_OWNER_ID`を削除（またはコメントアウト）して
  `sudo docker compose up -d butler-bot`
