# 13. ワニ博士タスク管理アプリ — あなたがやる作業

サーバー側は構築・起動済み。あなたの作業は「スマホにインストール」と
「GitHub連携の有効化」の2つ。**GitHubをやらなくてもモックデータで動く**ので、
まず1で触ってみてから2をやるのがおすすめ。

## 1. スマホ(Pixel 7a)にインストール — 3分

1. Pixel 7aの**TailscaleアプリをON**にする(最後の接続が古いので一度開いて接続)
2. Chromeで開く: `https://kokko-server-pavilion.tailed0412.ts.net:8443/`
3. ワニ博士が表示されたら、Chromeメニュー(︙) → **「ホーム画面に追加」→「インストール」**
4. ホーム画面の🐊アイコンから起動すると全画面のアプリとして動く

タスクの「未着手/進行中/完了」ボタンを押すと進捗が変わり、完了するとワニ博士が
喜ぶ。今はモックデータ(黄色い「モック」バッジが出ている状態)。

**UIは3モード**(上部タブで切替、選択は記憶される):
- **ぼうけん**: 射的屋台風の紙人形ステージ(横長なのでスマホ横向き推奨)。
  タスク=敵モンスター。敵をタップ→「完了」でとどめ(スロット演出)。
  タスクシート下段の「せんとう/まえへ/うしろへ」で戦う順を変更できる
  (GitHub Projectの並び順にも反映)。カレンダーの予定の時間になると
  時計モンスターが割り込むので、済んだら「すんだ！」で解散
- **リスト**: カード型の一覧
- **マップ**: すごろく型で今日の道のりを俯瞰

タスク追加はヘッダーの「＋」から(GitHubのDraft itemとして入る)。

**朝の作戦会議**: その日最初に開くと、Geminiが今日のカレンダーとタスク一覧から
「今日やる2〜4件」を理由付きで提案してくる。チェックを編集して
「これでスタート！」を押すと、ぼうけん/マップには今日のタスクだけが出る。
残りは脇道の「🗃 そのほか」に控えていて、タップ→「⭐きょうやるに入れる」で
いつでも隊列に呼び出せる。Discordで「今日は#10と#15やるわ」でも設定可能。

**ワニ博士を公式画像にしたい場合**(任意): 公式サイトから画像を保存して
`/mnt/data/ai/wani/assets/wani.png` に置くと、ぼうけんモードの紙人形が
その画像に自動で切り替わる(透過PNG推奨)。リポジトリに公式画像を
コミットしないこと(再配布になるため、各自ローカル配置のみ)。

※UIだけ先に見たい場合はPCでも同じURLでOK。デザイン確認用のモックは
Claude Artifactにも置いた(会話内のリンク参照。表情の切替デモ付き)。

## 2. GitHub連携を有効にする — 15分

guide/12(タスク管理エージェント)と共通の準備。**12を既にやったなら4だけでよい。**

1. **リポジトリとProject**: GitHubに`personal-tasks`リポジトリを作り、
   Projects(V2)を1つ作成。
   - **Projectの「デフォルトリポジトリ」設定は不要**(あれはProject画面から
     issueを作るときの既定先を決めるだけの便利設定で、API連携には関係ない)
   - Statusの選択肢は waiting / Todo / In Progress / Review / Done / wish list
     に対応済み。**アプリ側の扱い**: waitingとwish list(後回しBOX)は「今日の進捗」の
     分母から除外し、画面でも折りたたみ表示。Review移動も「前進」として気分が上がる
2. **タスクはissueで登録**し、Projectに追加する(スマホのGitHubアプリからも可)。
   **リポジトリに紐付かないDraft item(Projectで直接「+」から書くメモ)もOK**
   (アプリでは「メモ(Draft)」表示になり、Discordからはタイトル指定で操作する)
3. **Classic PAT発行**: GitHub → Settings → Developer settings →
   Personal access tokens → **Tokens (classic)** → Generate new token (classic)。
   スコープ: **repo** と **project** の2つにチェック。
   - **注意: fine-grained PATは使えない。** 個人アカウント直下のProject V2への
     アクセス権限がfine-grained PATには存在しない(Account permissionsにProjectsが
     出てこない)ため。GitHub側の未実装で、2026年6月時点でも未解決
     (community discussion #156512)。Organization配下のProjectなら
     fine-grained PATでも可
4. サーバーの **`~/dotfiles/server/.env`** (他のAPIキーと同じファイル)のGitHubセクションを記入:
   ```bash
   nano ~/dotfiles/server/.env
   ```
   - `GITHUB_TOKEN`: classic PAT(`ghp_`で始まる)
   - `GITHUB_PROJECT_OWNER`: あなたのGitHubユーザー名(または個人Organization名)
   - `GITHUB_PROJECT_OWNER_TYPE`: 個人アカウント直下のProjectなら `user`
   - `GITHUB_PROJECT_NUMBER`: ProjectのURL末尾の数字
   - **値の後ろに`#`コメントを書かないこと**(composeが値として読んでしまう)
5. 反映(restartでは.envを読み直さないので必ずこのコマンドで):
   ```bash
   cd ~/dotfiles/server
   docker compose up -d --force-recreate wani
   curl -s http://127.0.0.1:8090/api/state | head -5   # "mock": false になればOK
   ```
   アプリから「モック」バッジが消え、本物のissueが並ぶ。

## Discordから使う

butler(執事Bot)に話しかけるだけ:
- 「今日のタスクは？」→ 一覧とワニ博士の気分を報告
- 「◯◯終わったで」→ 該当タスクをDoneにしてワニ博士が喜ぶ
- 「◯◯はレビューまで進んだ」「◯◯は後回しで」なども通じる
  (waiting/Review/wish listへの移動に対応)

スマホアプリとDiscordは同じ状態を見ているので、どちらで更新しても揃う。

また、**毎朝8時のブリーフィング**にGoogleカレンダーの予定+タスク一覧+
「今日のおすすめタスク」(予定の空き具合を踏まえてGeminiが1〜3件提案)が含まれる。

## 困ったとき

- **スマホで開けない** → Tailscaleが接続中か確認。PCから
  `curl -s http://127.0.0.1:8090/healthz` が `{"ok":true}` ならサーバー側は正常
- **HTTPSの配信設定を確認** → `tailscale serve status` に
  `:8443 → 127.0.0.1:8090` の行があるか(サーバー再起動後も設定は保持される)
- **GitHub切替後にエラー** → `docker logs wani --tail 20`。PATの権限
  (Issues/Projects Read-Write)とProject番号を確認
- **気分をリセットしたい** → `sudo rm /mnt/data/ai/wani/wani_state.json` →
  `docker compose restart wani`


## Google ToDo連携(2026-07-11追加)

GoogleカレンダーのToDo(Google Tasks)もモンスターとして出せる:
- **期限付きToDo** → その日になると問答無用で隊列の**先頭**に出現(📝マーク・薄黄色の名前)
- **期限なしToDo** → 作戦会議で「今日やるか」選べる
- タップ→「⚔たおす(完了にする)」でGoogle側も完了になり、気分+18

**有効化に1回だけ必要な作業**(n8nにGoogle Tasksの認可を追加):
1. ブラウザで https://kokko-server-pavilion.tailed0412.ts.net (n8n)を開く
2. 左メニュー **Credentials** → **Add credential** → 「Google Tasks OAuth2 API」を検索
3. 既存のGoogle OAuthクライアント(Calendar連携で作ったもの)のClient ID/Secretを入れ、
   **Sign in with Google** で同意(Tasksスコープ)
4. サーバーで `cd ~/dotfiles/server && sudo python3 scripts/n8n_todo_workflows.py`
   を実行(ワークフロー2本が自動作成される)
5. `curl http://localhost:5678/webhook/todos` でToDoのJSONが返ればOK
