# 14. ワニ博士をスマホアプリにする(TWA)

PWAをTWA(Trusted Web Activity)でAPKに包んだ「ワニ博士」アプリ。
Playストアには出さず、自宅サーバーからのサイドロード限定
(ワニ博士は阪大の公式マスコットのため公開配布はしない)。

## インストール(Pixel 7a)

1. TailscaleをONにして、Chromeで開く:
   **https://kokko-server-pavilion.tailed0412.ts.net:8443/assets/wani.apk**
2. ダウンロード完了通知をタップ →「提供元不明のアプリ」の警告が出たら
   **設定 → この提供元を許可**(Chromeからのインストール許可) → **インストール**
3. ホーム画面に「ワニ博士」が追加される。起動するとURLバー無しの全画面アプリとして
   PWA(`https://…ts.net/wani/`)が開く

※以前の「ホーム画面に追加」版PWAが残っていたら消してよい(中身は同じ)。
※起動にはTailscale接続が必要(オフラインでは開けない)。

## 仕組み

- APKは中身を持たない薄いラッパーで、起動すると
  `https://kokko-server-pavilion.tailed0412.ts.net/wani/` を全画面Chromeで表示する。
  **UIの更新はサーバー側だけで完結**(APKの入れ直し不要)
- URLバー非表示の条件である Digital Asset Links は
  `https://…ts.net/.well-known/assetlinks.json` で配信(署名証明書のSHA256入り。
  tailscale serveのパスマウントで443の`/`はn8nのまま`/wani`と`/.well-known`だけ
  wani-apiに向けている)
- 署名キーストア: `/mnt/data/ai/wani/twa/wani.keystore`
  (パスワードは同じ場所の `keystore.pass`。**紛失すると同じIDで更新できなくなる**ので
  バックアップ対象に含めること。gitには入れない)

## APKを作り直すとき(バージョンアップ等)

```bash
cd ~/dotfiles/server/wani
./tools/build_apk.sh
```

- ビルド環境はdockerイメージ`wani-twa-builder`(JDK17+Android SDK+bubblewrap)。
  初回のみイメージ構築で数分かかる
- バージョンを上げる場合は `twa/twa-manifest.json` の appVersionName/appVersionCode を
  編集してから実行
- 出力: `/mnt/data/ai/wani/assets/wani.apk`(そのまま配信URLに反映される)。
  スマホ側は同じ署名なので上書きインストールできる

## トラブルシューティング

- **アプリ起動時に上部にURLバーが出る** → Digital Asset Linksの検証失敗。
  `curl https://…ts.net/.well-known/assetlinks.json` が署名フィンガープリント入り
  JSONを返すか確認。APKを作り直した(署名が変わった)場合はassetlinks.jsonも
  build_apk.shが再生成するが、Android側の検証キャッシュに数分かかることがある
- **「アプリはインストールされていません」** → 旧バージョンと署名が違う。
  旧アプリをアンインストールしてから入れ直す
