# 05. Immich 初期設定

## 概要

Immich は自己ホスト型の Google Photos 代替です。
スマホアプリから写真を自動バックアップして、ブラウザで閲覧できます。

---

## ステップ1: 管理者アカウントの作成

1. http://kokko-server-pavilion:2283 にアクセス（Tailscale接続必須）
2. **「アカウントを作成する」** 画面が表示される
3. 以下を入力：
   - 名前: 好きな名前
   - メール: 適当なメール（例: kokko@homeserver.local）
   - パスワード: 強いパスワード
4. **「サインアップ」** をクリック

> ⚠️ このアカウントが管理者になります。初回のみ作成画面が出ます。

---

## ステップ2: タイムゾーン設定

1. 右上アバター → **「管理パネル」**
2. **「設定」** → **「基本設定」**
3. タイムゾーン: `Asia/Tokyo` を選択
4. 保存

---

## ステップ3: スマホアプリのインストール

- iOS: App Store で「Immich」を検索
- Android: Play Store で「Immich」を検索

### アプリの設定

1. アプリを開く
2. サーバーURL: `http://kokko-server-pavilion:2283`（Tailscale接続中）
3. 先ほど作ったアカウントでログイン

---

## ステップ4: 自動バックアップの設定

アプリ下部の **「プロフィール」** → **「バックアップ」**：

| 設定 | 推奨値 |
|-----|------|
| バックアップ | ON |
| Wi-Fiのみ | ON（通信節約） |
| 充電中のみ | お好みで |
| ビデオ | ON（容量に余裕があれば） |

---

## ステップ5: 既存写真の確認

すでに `/mnt/data/immich` にデータがある場合、
**ライブラリ** → **外部ライブラリ** から既存データを読み込めます。

---

## 共有アルバムの作成（外部共有用）

写真を外部の人に共有したい場合：

1. アルバムを選択
2. **「共有」** → **「共有リンクを作成」**
3. 有効期限・パスワードを設定
4. リンクをコピーして相手に送る

> ⚠️ 有効期限は必ず設定しましょう（無期限リンクは避ける）

---

## メモリ節約（機械学習を無効化する場合）

`docker-compose.yml` で `immich-machine-learning` のプロファイルを有効化すると、
顔認識やCLIPが使えなくなる代わりにRAMを節約できます。

RAM不足でサーバーが重い場合：

```yaml
  immich-machine-learning:
    profiles: ["ml"]  # この行を追加すると通常起動から除外される
```

変更後:
```bash
sudo docker compose up -d
```
