# 15. 音楽サーバー設定（Navidrome）

手持ちの音楽をこのサーバーに置いて、**広告なし・無料**でスマホ／PCからストリーミング
再生する。Spotifyのような使い勝手を、自分の音源とTailscale経由の非公開アクセスで実現する。

- サーバー本体: **Navidrome**（Go製で軽量、Subsonic API 互換）
- 音源の置き場所: `/mnt/data/music`（Samba共有 `smb://kokko-server-pavilion/data/music` と同じ場所）
- アクセス: `http://kokko-server-pavilion:4533`（Tailscale経由のみ。外部には出さない）

---

## ステップ1: 起動

`server/.env` は Navidrome 用の秘密情報は不要（音源フォルダを既定の `/mnt/data/music`
以外にしたい場合だけ `MUSIC_LOCATION` を書く）。そのまま起動できる。

```bash
cd ~/dotfiles/server

# 音源フォルダを用意（初回のみ）
sudo mkdir -p /mnt/data/music
sudo chown $USER:$USER /mnt/data/music

# 起動
docker compose up -d navidrome

# ログ確認（"Navidrome server is accepting requests" が出ればOK）
docker compose logs -f navidrome
```

---

## ステップ2: 管理者アカウントを作る

1. ブラウザ（PC/スマホどちらでも、Tailscale ON）で開く:
   **http://kokko-server-pavilion:4533**
2. 初回アクセスで「管理者アカウント作成」画面が出る。**ユーザー名・パスワードを設定**する
   （このアカウントがスマホアプリのログインにも使う）。

---

## ステップ3: 音源を取り込む

Sambaファイル共有経由で音楽ファイルを `music` フォルダにコピーするだけ。

- **macOS**: Finder →「移動」→「サーバへ接続」→ `smb://kokko-server-pavilion/data`
  → `music` フォルダにドラッグ&ドロップ
- **Windows**: エクスプローラーのアドレスバーに `\\kokko-server-pavilion\data`
  → `music` へコピー
- **Linux**: `smb://kokko-server-pavilion/data`（接続手順は `guide/07_Samba設定.md`）

対応形式: mp3 / flac / m4a(AAC/ALAC) / ogg / opus / wav など。
フォルダ構成は自由だが、`アーティスト/アルバム/曲.mp3` のように整理し、ファイルに
タグ（アーティスト名・アルバム名・ジャケット画像）が埋まっていると綺麗に並ぶ。
ジャケットは各アルバムフォルダに `cover.jpg` を置いても認識する。

### スキャン反映

- 追加後、**最大1時間**で自動反映（`ND_SCANSCHEDULE=1h`）。
- すぐ反映したいときは Web UI 右上のメニュー →「管理」→ **今すぐスキャン**。

---

## ステップ4: スマホで聴く

Navidrome は Subsonic API 互換なので、専用の高機能アプリが使える。
どれも下記の共通設定でログインする。

| 項目 | 値 |
|-----|---|
| サーバー / アドレス | `http://kokko-server-pavilion:4533` |
| ユーザー名 | ステップ2で作ったもの |
| パスワード | ステップ2で作ったもの |

おすすめアプリ:

- **iOS**: Amperfy（無料）, play:Sub, substreamer
- **Android**: Symfonium（有料だが高機能）, Substreamer（無料）, DSub

※ 再生には **Tailscale をON**にしておくこと（自宅内・外出先とも Tailscale 経由で到達する）。
モバイル回線ではアプリ側の設定で低ビットレート再生（transcoding）を選ぶとデータ節約になる。

---

## トラブルシューティング

**曲が出てこない:**
- ファイルが `/mnt/data/music` 配下に入っているか（`ls /mnt/data/music`）。
- Web UI の管理画面から手動スキャン。`docker compose logs navidrome` にスキャン結果が出る。

**アプリからログインできない:**
- Tailscale が ON か、`tailscale status` でサーバーが Online か。
- アドレスは `http://`（`https`ではない）、ポート `4533` を含めているか。

**初回スキャンが重い / 落ちる:**
- 大量ライブラリの初回スキャンで一時的にメモリを使う。`docker-compose.yml` の
  navidrome の `mem_limit: 384m` を一時的に上げて再試行（RAM事情は `plan/09_improvements.md`）。

---

## 補足

- 音源フォルダは Navidrome から**読み取り専用**でマウントしている（`:ro`）。誤ってファイルを
  消したり書き換えたりしない。バックアップ対象は音源(`/mnt/data/music`)と設定DB
  (`navidrome-data` ボリューム)。`guide/10_バックアップ.md` の restic 対象に含めておくとよい。
- 外部ポート開放はしない方針（`plan/08_security.md`）。共有したい相手がいる場合のみ
  Tailscale の共有機能や `tailscale serve` を検討する。
