# ファイルサーバー

## 要件

- 個人用ファイル（写真メイン）の保管・アクセス
- アクセス元：Linux / スマホ（iOS or Android）
- 外部への限定リンク共有（写真アルバム等）
- シングルユーザー運用

## メインストレージ

`/dev/sdb`（1.8 TB）を `/mnt/data` にマウントして全サービスで共用する。

```bash
# /etc/fstab に追加
UUID=<sdb-uuid>  /mnt/data  ext4  defaults,nofail  0  2
```

### ディレクトリ構成

```
/mnt/data/
├── photos/          # Immich の元データ（ライブラリ）
├── documents/       # 一般ファイル・研究資料など
├── shared/          # 外部共有候補ファイルの一時置き場
├── ai/              # AIエージェントのワークスペース
│   └── context/     # 執事が参照するメモ・カレンダーキャッシュなど
└── backup/          # バックアップ受け口（将来）
```

## サービス構成

### Immich（写真管理）

写真・動画の管理に特化した自己ホスト型 Google Photos 代替。
スマホアプリからの自動バックアップ、アルバム共有リンク機能を持つ。

| 項目 | 内容 |
|-----|------|
| ポート | 2283 |
| データ保存先 | /mnt/data/photos |
| アクセス | Tailscale 経由（http://server:2283）|
| 外部共有 | アルバム共有リンク機能を使用（パスワード設定可）|
| スマホ連携 | Immich アプリで自動アップロード |

### Samba（汎用ファイル共有）

Linux / macOS 向けのファイル共有。写真以外のドキュメントへのアクセスに使う。

| 項目 | 内容 |
|-----|------|
| 共有名 | `data` |
| 共有パス | /mnt/data |
| アクセス | 自ユーザーのみ（認証あり）|
| ゲストアクセス | 無効 |

```ini
# smb.conf の主要設定（抜粋）
[data]
  path = /mnt/data
  valid users = kokko
  read only = no
  browseable = yes
```

## 外部共有フロー

```
1. 共有したいファイルを /mnt/data/shared/ に置く
2. Immich の場合：アルバムから共有リンク生成（有効期限・パスワード設定）
3. ドキュメントの場合：別途 nginx + 一時トークン方式を検討（Phase 2 以降）
```
