# 14. Discord Bot 経由での日記機能

## 1. 概要
Discord のプライベートチャンネルへの投稿を、日記として半永久的に記録する機能。
ユーザーは手軽に日々の出来事や感情を記録でき、AIはその記録を後から検索・分析しやすい構造化データとして保存する。

## 2. ユーザー要件
- **入力:** 特定のDiscordチャンネルへの自由なテキスト投稿。
- **処理:** 投稿されたテキストをLLMで解析し、メタデータを付与して保存。
- **保存形式:** AIが後から検索・分析しやすい形式（JSON Lines）。
- **検索:** （将来的な展望）「あの時どう思ってたっけ？」といった自然言語での検索に対応できるようにする。

## 3. 実装計画

### 3.1. 保存形式の決定
- **ファイル形式:** JSON Lines (`.jsonl`) を採用。1行1JSONオブジェクトの形式で、追記が容易かつ、大量データの扱いに適している。
- **ディレクトリ構成:** `/mnt/data/ai/diary/YYYY/YYYY-MM.jsonl` のように、年・月でファイルを分割する。
- **データ構造（1行のJSONオブジェクト）:**
  ```json
  {
    "id": "DiscordのメッセージID",
    "timestamp_utc": "ISO 8601形式のタイムスタンプ (UTC)",
    "author_id": "DiscordのユーザーID",
    "content": "投稿された日記の本文",
    "llm_analysis": {
      "summary": "LLMが生成した一行要約",
      "tags": ["仕事", "プログラミング", "悩み"],
      "emotion": "ポジティブ | ネガティブ | 中立"
    }
  }
  ```

### 3.2. `butler-bot` の改修
- **対象ファイル:** `server/ai/src/bot.py`
- **改修内容:**
  1.  `on_message` イベントハンドラを修正。
  2.  環境変数 `DIARY_CHANNEL_ID` を新設し、そのチャンネルIDからの投稿のみを対象とするロジックを追加。
  3.  対象チャンネルへの投稿があった場合、以下の処理を実行する。
      a. 投稿内容 (`message.content`) を取得。
      b. LLM (Gemini) に `content` を渡し、「この日記を1行で要約し、関連するタグを5つ生成し、全体の感情をポジティブ・ネガティブ・中立のいずれかで判定してください。結果はJSONで返してください。」といったプロンプトで解析を依頼。
      c. LLMからのJSONレスポンスと、元のメッセージ情報 (`id`, `created_at`, `author.id`) を組み合わせて、上記のデータ構造のPython辞書を作成。
      d. 保存先のパスをタイムスタンプから決定 (`/mnt/data/ai/diary/YYYY/YYYY-MM.jsonl`)。
      e. ファイルを追記モード (`'a'`) で開き、Pythonの `json.dumps()` でシリアライズした文字列を1行として書き込む。

### 3.3. 設定
- **`.env.example`:** `DIARY_CHANNEL_ID=` を追加。
- **`docker-compose.yml`:** `butler-bot` サービスに、`/mnt/data/ai/diary:/app/diary` のようなボリュームマウントを追加する。

## 4. 今後の展望
- **日記検索機能:** `/search_diary <キーワード>` のようなコマンドで、保存した `.jsonl` ファイル群を横断的に検索し、関連する日記を要約して提示する機能。
- **週次・月次サマリー:** 定期的に `major_events` のように、日記の内容をLLMが要約し、自動で月報を生成する機能。
