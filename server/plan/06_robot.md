## 通信アーキテクチャ

### 第一候補：Wi-Fi 経由 ROS2

```
[AIエージェント（サーバー）]
    │
    │  HTTP / WebSocket（rosbridge）
    ▼
[rosbridge_suite（サーバー or ロボット側）]
    │
    │  ROS2 トピック / アクション
    ▼
[ロボット本体（ROS2ノード群）]
    ├── navigation stack
    ├── actuator drivers
    └── sensor nodes
```

### フォールバック：ESP32 シリアル受信

Wi-Fi ROS2 が難しいマイコンベースの構成の場合：

```
[AIエージェント] → HTTP → [サーバーのブリッジスクリプト] → UART/TCP → [ESP32/Arduino]
                                                                           │
                                                                        モーター・サーボ
```

ESP32 側はシンプルなコマンドプロトコル（例：`MOVE:FORWARD:0.5`）をデコードするだけ。

## AIエージェント → ロボット制御フロー

```
ユーザー: 「玄関に行って」
    │
    ▼ LLM（高レベル命令を解析）
目標: 玄関座標 (x: 1.2, y: 0.3)
    │
    ▼ rosbridge WebSocket
ROS2 Action: /navigate_to_pose
  goal.pose.x = 1.2
  goal.pose.y = 0.3
    │
    ▼ ロボット Navigation Stack
実行
```

LLM の役割は「抽象命令 → 具体的なROS2コマンド」への変換。
ロボット側の実装は最小限で良い（標準的なNav2スタックを使えばよい）。

## ROS2 バージョン互換性問題

現サーバーは Ubuntu 26.04 LTS。ROS2 のサポート状況：

| ROS2 ディストリビューション | Ubuntu | EOL |
|--------------------------|--------|-----|
| Humble | 22.04 | 2027-05 |
| Iron | 22.04 | 2024-12 |
| Jazzy | 24.04 | 2029-05 |
| Kilted | 24.04 | 2026-11 |
| Rolling | 最新 | - |

Ubuntu 26.04 対応の安定版は現時点で存在しない可能性がある。

### 対応方針

| 選択肢 | 内容 | 推奨度 |
|------|------|------|
| **Docker コンテナ内で ROS2** | サーバー上の Ubuntu 26.04 に影響を与えずに Jazzy を動かす | ◎ |
| ロボット本体に Ubuntu 24.04 | ロボット側の計算機（Raspberry Pi 等）で Jazzy を動かす | ◎ |
| Rolling を使う | 最新だが不安定、サポートなし | △ |

→ **サーバー上の rosbridge は Docker コンテナ内（Ubuntu 24.04 + ROS2 Jazzy）で動かす**方針を推奨。
　 ロボット本体の計算機も Ubuntu 24.04 で揃えると管理が楽。
