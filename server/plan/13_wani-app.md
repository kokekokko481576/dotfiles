# 13. ワニ博士タスク管理アプリ

たまごっち風のドット絵キャラ「ワニ博士」(大阪大学マスコット)が、その日のタスク進捗で
元気になったりぐったりしたりするタスク管理アプリ。タスクの正はGitHub Projects V2
(12_task-management.mdと同じProject)で、スマホ(PWA)とDiscord(butler-bot)の両方から
進捗を更新できる。

## 構成

```
[Pixel 7a PWA] --Tailscale HTTPS(8443)--> [wani コンテナ :8090]
[Discord] --> [butler-bot] --compose内HTTP--> [wani コンテナ :8090]
                                                |
                                                +-- GitHub Projects V2 (GraphQL)  ←タスクの正
                                                +-- /mnt/data/ai/wani/*.json      ←気分・履歴・モック
```

- **wani** (`server/wani/`): FastAPI。REST API + PWA静的配信を1コンテナで担う。
  - `GET /api/state` — 気分・タスク一覧・進捗
  - `POST /api/tasks/{item_id}/status` — Status更新(気分イベントも発火)
  - GitHub未設定(`GITHUB_TOKEN`空)なら**モックモード**で動く。環境変数は
    `task-agent/.env`をそのまま共用(設定箇所を1つにするため)。
- **butler-bot**: `list_tasks` / `update_task_status` ツールを追加。会話から
  「これ終わったで」→ Done化 → ワニ博士の気分報告、ができる。
- 状態はすべてwani APIに集約。**PWAとDiscordのどちらから更新しても同じ気分が動く。**

## Status体系(ユーザーの実Projectに準拠)

waiting / Todo / In Progress / Review / Done / wish list の6種。
- **waiting**(他人待ち)と**wish list**(後回しBOX。入ると無限に遅延されがち、と
  ユーザー本人談)は「今日の進捗」の分母から除外し、UIでは折りたたみグループ表示
- Projectのデフォルトリポジトリ設定は不要。リポジトリに属さないDraft itemにも対応
  (番号がないため、Discordからはタイトル部分一致で指定する)

## 気分エンジン (`wani/src/mood.py`)

- 気分は0-100の連続値。Done +18 / In Progress・Review +6 / Doneから戻す -10
- 起床時間帯(7-24時)に1.5/時で減衰。0-7時は睡眠(減衰なし・寝スプライト表示)
- レベル: excellent(≥75) / happy(≥45) / normal(≥20) / tired(<20)
- streak: 1件以上Doneにした連続日数
- パラメータは全部このファイルの定数。飽きたら調整する

## UIモード(3種、タブ切替。選択はlocalStorageに保存、?mode=xxxで直接指定可)

1. **ぼうけん**(デフォルト) — 縁日の射的/アーケード筐体風の「紙人形」ステージ
   (640x220の横長。スマホは横向き推奨)。全キャラは下から棒で突き出したパペットで、
   ゆれ=棒先端を支点の微回転、攻撃/被弾=紙をパタッと裏返すフリップ(裏面は別の表情)、
   討伐=棒ごと下に引っ込む。ワニ博士はドット絵ではない滑らかなカートゥーン一枚絵
   (canvasパスで自作。**公式画像を使いたい場合は/mnt/data/ai/wani/assets/wani.pngに
   置けば自動で差し替わる**。再配布回避のためリポジトリには含めない)。
   - アクティブなタスク=敵の隊列。**並び順はGitHub Projectの手動順**で、
     **敵を直接ドラッグ**またはタスクシートのボタンで変更できる
     (updateProjectV2ItemPosition、GitHub側のボードにも反映)。
     タップ=シート/横ドラッグ=並び替えを12px閾値で判別。画面外の敵は「→あと N体」表示。
     画面はHUD(📋作戦会議ボタン含む)+キャンバス+メッセージ窓のみで、
     待ち/後回し等の管理はリストモードに集約。
     ワニ博士は**右斜め向きのドット絵**(`side_*`スプライト、リスト表示と同一キャラの
     3/4ビュー、5表情+まばたき)。/mnt/data/ai/wani/assets/wani.pngを置けば
     その画像に差し替わる(非同梱、guide/13参照)。
   - HPバーが状態(Todo満タン→In Progress 2/3→Review 1/3)。完了=とどめ(スロット演出、
     3割でかいしんのいちげき)→討伐→ドロップ→前進。着手=ツボ割り。夜(0-7時)は野営。
   - **Googleカレンダー連携**: 予定の開始時刻に「じかんまじん」(時計モンスター)が
     **右上から飛来して先頭の敵の手前上空でホバリング、終了時刻に自動で消える**
     (操作不要。タップすると残り時間をメッセージ表示)。
     予定はwani-apiがn8n Webhook経由で取得(5分キャッシュ)。
   待ち/後回しは道の脇(⛺/🧰)に折りたたみ。
2. **リスト** — コンパクトな1行1タスク表示(行タップで共通タスクシート)。
   表示順は「ぼうけん同期(=Projectの手動順、デフォルト)/期限順/番号順」を切替可。
   期限はProjectの日付フィールド(名前不問、最初の1つ)を`due`として取得し、
   当日=黄・超過=赤のチップ表示。**期限チップをタップするとその場で編集・クリア**
   できる(`POST /api/tasks/{id}/due`、GitHubの日付フィールドに反映)。
   **ぼうけん同期のときは≡ハンドルのドラッグで並び替え**でき、
   GitHubのボード順にも反映される。
3. **マップ** — すごろく型の俯瞰マップ。討伐済みは旗、**現在地のマスはワニ博士の
   顔ドット絵で埋まる(すごろくのコマ)**、最後に🏰。

演出はデータと分離しており、スロットの目は完了操作の結果に影響しない(見た目だけ)。
`prefers-reduced-motion`ではアニメーションを止めて静止画描画になる。

## 朝の作戦会議(今日やるリスト)

その日最初にアプリを開くと「作戦会議」シートが自動で開き、2〜4件を理由付きで提案する
(`POST /api/today/recommend`)。チェックボックスで編集して「これでスタート！」で確定
(`POST /api/today`)。提案の出どころは次の優先順で、いずれも大日程(`major_events`)・
プロフィール・週次蒸留した学習メモ(`learned_prefs.md`)を前提にしている:
1. **🌙夜間プラン**: task-agentが毎朝4時にGemini(LiteLLM経由)で生成した当日の
   `daily_plan.json`(item_id付き)をそのまま表示。待ち時間ゼロで、大日程・学習メモ込みで
   前もって選んだもの(繁忙期は0件のこともある)。LiteLLM不調時はOllamaにフォールバック。
2. **✨ライブGemini**: 夜間プランが無い/古い/現在のタスクに解決できないとき、その場で
   LiteLLM経由のGeminiが生成(着手済みの完遂優先・予定との相性を考慮)。
3. **ヒューリスティック**: LLM未設定/失敗時は先頭から3件。
- 承認後、**ぼうけん/マップは今日のリストだけ**を表示。それ以外のアクティブタスクは
  脇道「🗃 そのほか」から**「⭐きょうやる」でいつでも隊列に追加できる**(外すのも可)
- リストは`/mnt/data/ai/wani/today.json`に保存、日付が変わると自動リセット
- Discordからも「今日は#10と#15やる」で設定できる(`set_today_tasks`ツール)
- 承認した内容は翌朝task-agentが提案実績と突き合わせて学習メモに反映する
  (詳細は`guide/12`の「レコメンドの自己強化ループ」)

## タスク追加

- PWA: ヘッダーの「＋」→ タイトル入力のみの控えめなシート
- Discord: 執事に「〜をタスクに入れといて」(`create_task`ツール)
- 実体はGitHub Projectの**Draft item**として追加(リポジトリ不要)、Status=Todo

## ドット絵 (`wani/tools/gen_sprites.py`)

32x32ピクセル、5表情(normal/happy/excellent/tired/sleeping)×各2-4フレーム。
公式マスコットの見た目(黄色い目・クリーム腹+横線・白爪牙・ピンクの口)準拠。
敵モンスター5種+ツボ(通常/割れ)+薬草+コインも同ファイルで管理(`OBJECTS`)。
ASCIIグリッドが原本で、実行すると`static/sprites.js`とPWAアイコンPNGを再生成する
(依存ライブラリなし)。キャラを差し替えたいときはこのファイルだけ編集すればよい。

## プレビュー生成 (`wani/tools/build_preview.py`)

実アプリのJSをそのまま連結しfetchだけモックに差し替えた単一HTMLを生成する
(Claude Artifact用)。実装とプレビューの乖離を防ぐため手書き複製は持たない。

## Androidアプリ化 → **TWAで実装済み**(guide/14)

bubblewrap(docker化: `wani/twa/Dockerfile`)でPWAをAPKに包んだ。
`tools/build_apk.sh`一発でビルド〜assetlinks生成〜配信配置まで完了する。
- パッケージ: dev.kokko.wani / 表示名「ワニ博士」/ サイドロード限定(Play配布なし)
- 起動URL: https://…ts.net/wani/ (443のパスマウント。UIの更新はサーバー側のみで完結)
- 署名キーストアは/mnt/data/ai/wani/twa/(git外、keystore.passと共にバックアップ必須)
- さらにネイティブ寄りにしたくなったら(通知・ウィジェット等)はCapacitor/Kotlinを検討

## 収益化について(ユーザーの質問への回答)

- **ワニ博士のままでは不可。** ワニ博士は大阪大学の公式マスコットで著作権・商標は
  大学側にある。個人利用の範囲なら実害は考えにくいが、配布・販売は許諾が要る。
  リリースするならキャラをオリジナルに差し替える(gen_sprites.pyの編集だけで済む設計)。
- 「タスク完了でキャラが育つ」系アプリはHabitica、Finch等の先行が強い。差別化するなら
  「GitHub Projects直結」「自宅サーバー完結でデータが手元に残る」という開発者向けの
  切り口が現実的。まずは自分が毎日使うものに育てるのが先。
