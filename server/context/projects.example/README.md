# 進捗ステート (managed projects)

「GitHubのissueにはしないが管理対象」の領域（院試勉強・研究・バイト等）の進捗を置く場所。

- **実体の置き場所:** `/mnt/data/ai/context/projects/*.md`（サーバー上、**git管理外**）。
  docker-composeで各Botに `/app/context/projects/` として見える。
- **役割:** 夜間の日次プラン生成(`task-agent daily-plan`)がここを読み、「次にやるべき具体タスク」を
  **創出**する（過去問分析・得点率・学習到達度・弱点などから）。
- **更新方法:** 2通り。
  1. 執事Discord botに話す（例: 「過去問2020の電磁気7割だった」「流体の教科書3章まで終わった」）
     → botが該当ファイルに追記/更新する（Phase 2の `update_progress` ツール）。
  2. このmarkdownを直接手編集する。
- **書式:** 自由記述でよいが、下の `院試.md` のように「頻出分析 / 得点率 / 到達度 / 弱点 /
  次やるべき候補」があるとLLMが具体タスク化しやすい。

このディレクトリ(`projects.example/`)はgitに置くテンプレート。実運用のファイルは
上記サーバーパスに置く（中身は個人データなのでgitに入れない）。
